# Как добавлять материалы, продукты и формы

Инструкция для того, кто продолжит работу над порталом. Три частых задачи:
добавить **материал** в каталог, добавить **продукт** (курс/лендинг) и добавить
**форму/раздел**.

---

## 1. Добавить материал в каталог «Разборы райта»

### Способ А — через админку (обычный путь)
1. Откройте `/admin/materials`.
2. Заполните форму «Добавить материал»:
   - **Заголовок** — как будет в карточке.
   - **Тип** — статья / видео / чек-лист / разбор магазина / пост.
   - **Категория** — из списка (список категорий см. ниже).
   - **Краткое описание** — 1–2 предложения для карточки.
   - **Текст материала** — необязательно (для статей и чек-листов).
   - **Ссылка на материал** — если контент внешний (видео, PDF, страница).
   - **Ссылка на обложку** — если пусто, рисуется градиентная заглушка.
   - **Бесплатный материал** — галочка (влияет на пометку в карточке).
3. Нажмите «Добавить материал». Он сразу появится на главной (каталоге) и на
   `/admin/materials`.

> `slug` (адрес страницы `/razbory/<slug>`) генерируется автоматически из
> заголовка (транслитерация). Уникальность гарантируется.

### Способ Б — вручную в JSON (если нужно массово)
Откройте `portal/data/materials.json` и добавьте объект в массив `materials`:
```json
{
  "id": "m-010",
  "slug": "moy-material",
  "type": "article",
  "title": "Заголовок",
  "description": "Краткое описание для карточки.",
  "category": "stati",
  "coverUrl": "",
  "contentUrl": "",
  "body": "Полный текст. Переносы строк = абзацы.",
  "isFree": true,
  "createdAt": "2026-07-14T10:00:00.000Z"
}
```
Поле `category` должно совпадать со `slug` существующей категории.

## 2. Добавить категорию каталога

В `portal/data/materials.json` в массив `categories` добавьте:
```json
{ "slug": "novaya-kategoriya", "title": "Новая категория" }
```
После этого категория появится в фильтрах на главной и в выпадающем списке
админки. `slug` — латиницей, без пробелов.

---

## 3. Добавить продукт (курс / лендинг / внешнюю ссылку)

Все внешние адреса лежат в одном файле — `portal/lib/links.ts`.
1. Добавьте ссылку в объект `externalLinks`:
   ```ts
   export const externalLinks = {
     // ...
     newProduct: "https://romarayt.ru/new",
   } as const;
   ```
2. Используйте её в нужном месте (шапка `components/SiteHeader.tsx`, подвал
   `components/SiteFooter.tsx`, страница `/kursy` и т.д.):
   ```tsx
   import { externalLinks } from "@/lib/links";
   <a href={externalLinks.newProduct} target="_blank" rel="noopener">Новый продукт</a>
   ```

> Готовые квиз и лендинги **не пересобираются** — портал только ссылается на них.
> Поменялся адрес продукта — правится один файл `lib/links.ts`.

## 4. Добавить форму или новый раздел

### Новый раздел-страница
1. Создайте папку в `portal/app/<имя-раздела>/` и файл `page.tsx`.
2. Для «заготовки в разработке» используйте готовый компонент:
   ```tsx
   import StubBanner from "@/components/StubBanner";
   export default function Page() {
     return (
       <div className="container">
         <StubBanner title="Название раздела">Описание.</StubBanner>
       </div>
     );
   }
   ```
3. Добавьте ссылку на раздел в навигацию (`components/SiteHeader.tsx`).

### Форма, которая что-то сохраняет
1. Клиентский компонент формы кладите в `portal/components/` (пример —
   `MaterialForm.tsx`).
2. Обработчик — в `portal/app/api/<имя>/route.ts` (пример —
   `app/api/materials/route.ts`): валидация + запись через слой данных.
3. Данные — только через слой `portal/lib/` (не пишите в файлы/БД напрямую из
   страниц), чтобы позже легко заменить хранилище.

---

## 5. Переход с демо-JSON на настоящую базу (Prisma)

Сейчас данные каталога — в `data/materials.json`, доступ изолирован в
`lib/materials.ts`. Когда появится доступ к базе:
1. Подключите Prisma, опишите модели `Material` и `Category` (поля уже совпадают
   с типами в `lib/types.ts`).
2. Перепишите тело функций в `lib/materials.ts` на вызовы Prisma
   (`prisma.material.findMany()` и т.д.).
3. Страницы и компоненты менять **не нужно** — они зависят только от этих функций.
