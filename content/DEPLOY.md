# Публикация портала на Vercel + Upstash Redis

Полноценный прод: сайт всегда онлайн, **заявки и правки контента из админки
сохраняются в базе** (Upstash Redis) и переживают перезапуски и передеплои.

Почему Redis, а не файлы: на Vercel файловая система read-only — писать в
`data/*.json` нельзя. Все хранилища (заявки, каталог, редактор контента лендингов,
быстрые правки) ходят через `lib/store.ts`, который на проде использует Redis, а
локально — те же файлы `data/*.json`. Код при переключении не меняется — только env.

---

## Что понадобится (аккаунты, всё есть с бесплатным тарифом)
- **GitHub** — залить код.
- **Vercel** — хостинг Next.js (вход через GitHub).
- **Upstash Redis** — база. Проще всего подключить **изнутри Vercel** (Marketplace),
  тогда ключи добавятся в проект автоматически.

---

## Шаг 1. Залить код на GitHub
Репозиторием делаем **саму папку `portal/`** (внутри неё уже есть `.gitignore`).

```bash
cd "portal"
git init
git add .
git commit -m "Портал Райта: воронка + админка + Redis-хранилище"
```
Создайте пустой репозиторий на github.com (например `rayt-portal`, private) и:
```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/rayt-portal.git
git branch -M main
git push -u origin main
```
> `.env.local` и локальные данные (`leads.json`, `site-content.json`, `landing-*.json`)
> в репозиторий НЕ попадут — они в `.gitignore`. `data/materials.json` (сид каталога)
> попадёт — так и надо.

## Шаг 2. Импортировать проект в Vercel
1. vercel.com → **Add New… → Project** → выберите репозиторий `rayt-portal`.
2. Framework Preset: **Next.js** (определится сам).
3. **Root Directory**: оставьте `./` (репозиторий и есть папка portal).
4. Пока **не жмите Deploy** — сначала база и переменные (Шаги 3–4). Если уже задеплоили —
   ничего страшного, после настройки env нажмите **Redeploy**.

## Шаг 3. Подключить базу Upstash Redis (авто-ключи)
1. В проекте Vercel откройте вкладку **Storage** (или **Integrations → Marketplace**).
2. Найдите **Upstash for Redis** → **Add / Connect** → создайте базу (регион ближе к
   пользователям, напр. `eu-central`) и **привяжите к проекту**.
3. Vercel сам добавит в проект переменные `UPSTASH_REDIS_REST_URL` и
   `UPSTASH_REDIS_REST_TOKEN`. Вручную вписывать ничего не нужно.

> Альтернатива без Marketplace: завести базу на upstash.com → в разделе **REST API**
> скопировать `UPSTASH_REDIS_REST_URL` и `UPSTASH_REDIS_REST_TOKEN` и вписать их
> руками в Шаге 4.

## Шаг 4. Задать переменные окружения
Vercel → проект → **Settings → Environment Variables** (среда **Production**, а лучше
и Preview). Добавьте:

| Переменная | Значение | Обязательна |
|---|---|---|
| `OPENAI_API_KEY` | ваш ключ OpenAI | да (для ИИ-разбора квиза) |
| `ADMIN_USER` | логин админки | да |
| `ADMIN_PASSWORD` | **новый** пароль админки | да |
| `UPSTASH_REDIS_REST_URL` | добавит Upstash-интеграция | да (для сохранения данных) |
| `UPSTASH_REDIS_REST_TOKEN` | добавит Upstash-интеграция | да |
| `TELEGRAM_BOT_TOKEN` | токен бота от @BotFather | желательно |
| `TELEGRAM_CHAT_ID` | ваш chat_id (напр. @userinfobot) | желательно |
| `LEADS_WEBHOOK_URL` | URL вебхука (Sheets/CRM) | нет |

Полный список с пояснениями — в `.env.example`.

## Шаг 5. Задеплоить и проверить
1. **Deployments → Redeploy** (или новый пуш в `main`).
2. Откройте выданный домен `https://ваш-проект.vercel.app`:
   - главная, `/quiz`, лендинги открываются, стиль тёмный/розовый;
   - пройдите квиз, оставьте заявку → должна прийти в Telegram и появиться в
     `/admin/leads` (данные в Redis, не в файле);
   - зайдите в `/admin` (логин/пароль из env) → **Контент страниц** → измените текст
     лендинга → **Сохранить** → обновите страницу лендинга: правка на месте и
     **сохраняется после передеплоя** (это и есть эффект Redis).
3. Заявки/правки видны и сохраняются между заходами — значит база работает.

---

## Своё доменное имя (по желанию)
Vercel → проект → **Settings → Domains** → добавьте домен (напр. `portal.romarayt.ru`)
и пропишите DNS-запись, которую покажет Vercel. HTTPS-сертификат Vercel выпустит сам.

## Обновления сайта после деплоя
Любой `git push` в `main` → Vercel пересобирает и публикует автоматически.
Контент лендингов/заявки при этом **не сбрасываются** (лежат в Redis, отдельно от кода).

## Важные заметки
- **Смените пароль админки** (`ADMIN_PASSWORD`) перед публикацией — дефолт демо небезопасен.
- **OpenAI**: каждый прогон квиза расходует кредиты вашего ключа. Следите за лимитами
  в OpenAI (billing → usage limits).
- **Сид каталога**: при первом деплое (пустой Redis) каталог показывает демо-материалы
  из `data/materials.json`. Как только вы добавляете/удаляете материал в админке — данные
  начинают жить в Redis. Чтобы поменять стартовый набор — правьте `data/materials.json`
  и пушьте (влияет только пока в Redis ещё нет ключа `materials`).
- **Бэкап данных**: в консоли Upstash можно смотреть/экспортировать ключи
  (`leads`, `materials`, `site-content`, `landing:*`).

---
Хранилище-абстракция: `lib/store.ts` (`kvGet`/`kvSet`/`kvDel`). Стор-модули
(`lib/leads.ts`, `lib/materials.ts`, `lib/siteContent.ts`, `lib/landingContent.ts`)
ходят только через неё — при желании поменять Redis на другую БД правится один файл.
