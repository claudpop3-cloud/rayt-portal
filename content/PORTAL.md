# Портал Makeunion — ТЗ и карта структуры

> Чистая версия ТЗ портала (Проект 01), по которой собран этот проект.
> Источник — видео-бриф заказчика (Рома Райт) и материалы архива в корне репозитория.

## 1. Что это за проект
Портал — **единая точка входа** («обёрочный» слой) над существующими продуктами:
квиз, лендинги сопровождения, курсы и каталог материалов. Не отдельный сайт, а
оболочка, которая ведёт разные аудитории по их путям. Готовые продукты **не
пересобираются** — подключаются ссылками.

**Приоритет заказчика:** главная страница = **каталог материалов** (основной экран).

## 2. Аудитории и пути
- **Селлер** — продаёт/хочет продавать на Ozon: Главная → Квиз → сопровождение или курс.
- **Менеджер** — ведёт чужие магазины: Главная → Квиз → сопровождение менеджера или курс.
- **Ученик / не определился** — Главная → Каталог материалов (бесплатные разборы, чек-листы).

Квиз — общий вход, квалифицирует и разводит по путям.

## 3. Карта разделов
**Публичная часть:**
1. Главная (= каталог материалов + компактные входы по ролям).
2. Квиз — внешняя ссылка (`my.makeunion.me/simulators/ozon-quiz`).
3. Сопровождение · Селлер — внешняя ссылка (`romarayt.ru/sopform`).
4. Сопровождение · Менеджер — внешняя ссылка (`romarayt.ru/sopmanager`).
5. Курсы · Селлер/Менеджер — внешние ссылки (`romarayt.ru/ozon`, `/manager`).
6. Каталог «Разборы райта» — статьи, видео, чек-листы, разборы. Без регистрации.
7. Финальная «Спасибо» — внешняя (`romarayt.ru/sopfinal`); в портале — заготовка.

**Админ-часть (CMS):**
- Управление материалами (реализовано: `/admin/materials`).
- Далее: управление пользователями, заявки квиза + экспорт, аналитика квиза,
  промо-блоки — заготовки на будущее.

## 4. Что собрано в этом проекте (тестовая неделя)
Полноценно:
- **Главная = каталог** — входы по ролям + фильтр по категориям + карточки.
- **Карточка материала** `/razbory/[slug]`.
- **Упрощённая админка** `/admin/materials` — добавление материала (форма + API).

Заготовки «в разработке»: `/kursy`, `/final`, блоки аналитики/заявок.
Инструкция по добавлению — `content/DOBAVLENIE-MATERIALOV.md`.

## 5. Единый стиль (по решению заказчика)
- Шрифты: **Dela Gothic One** (заголовки) + **Golos Text** (текст).
- Цвета: `#ff006b` (розовый), `#591ba1` (фиолетовый), `#fa003a` (красный),
  градиент `linear-gradient(135deg,#591ba1,#ff006b,#fa003a)`, тёмный фон.
- Стиль эталона `my.makeunion.me` (единый с доработанным квизом).
- Токены — в `portal/app/globals.css`.

> NB: в архиве `design-guide-v2.md` описывает более ранний вариант (Inter/#EC4899).
> По прямому указанию заказчика используется вариант из этого ТЗ.

## 6. Стек и правила
- **Next.js (App Router) + TypeScript**, деплой Vercel.
- Готовые квиз/лендинги — только ссылки (`portal/lib/links.ts`).
- Данные каталога — демо-JSON (`data/materials.json`), доступ изолирован в
  `lib/materials.ts` под будущий Prisma+БД.
- Секреты — только в `.env.local`, не в коде.

## 7. Открытые вопросы к заказчику
- Логотип и аватар (`logo.svg`, `roma-avatar.png`) — сейчас плейсхолдеры.
- Точный публичный URL квиза (взят `my.makeunion.me/simulators/ozon-quiz`).
- Доступ к базе материалов (`base.romarayt.ru`) — для замены демо-данных.
- Домен и аккаунт Vercel — на этапе деплоя.
