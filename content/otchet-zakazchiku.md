# Отчёт заказчику — Проект 01 (Портал по Ozon и маркетплейсам)

> Готов к отправке в рабочий чат. Правьте под себя.

---

**Проект:** №1 — Корпоративный портал по Ozon и маркетплейсам.

**Почему выбрал:** это задача про архитектуру и структуру — как собрать разрозненные продукты (квиз, лендинги, курсы, материалы) в единую точку входа и провести по ней разные аудитории. Мне ближе всего собирать понятную структуру и рабочий результат, а не только визуал, поэтому взял её.

**С чего начал:** изучил присланный архив (квиз, прототипы лендингов, дизайн-гайд, ТЗ-документы) и собрал из него карту структуры портала — разделы, пользовательские пути (селлер / менеджер / ученик), приоритеты. Стек — Next.js (App Router) + TypeScript, как основа под дальнейшее развитие и деплой на Vercel.

**Что уже готово (можно смотреть):**
- **Главная = каталог материалов** «Разборы райта» — основной экран: входы по ролям + фильтр по категориям + карточки. (Приоритет, который вы обозначили.)
- **Страница материала** — открывается по клику из каталога.
- **Простая админка** — добавление материала (тип, категория, заголовок, ссылка/текст); новый материал сразу появляется в каталоге.
- **Левое меню-навигация** по всем разделам.
- **Заготовки** остальных разделов (курсы, финальная) — понятные, помечены «в разработке».
- **Инструкция** «как добавлять материалы, продукты и формы» — отдельным документом.
- **Единый стиль** — снял его напрямую из вашего квиза-эталона (тёмная тема, Golos Text, фирменный градиент), чтобы портал выглядел заодно с квизом.

**Что НЕ трогал (подключено ссылками, как договаривались):** квиз, лендинги сопровождения, курсы — ведут на ваши живые страницы, не пересобирал.

**Данные:** каталог сейчас на демонстрационных материалах. Структуру заложил так, чтобы без переделки перейти на реальную базу.

**Вопросы к вам (что нужно, чтобы довести до конца):**
1. Доступ к базе материалов (`base.romarayt.ru`) — чтобы заменить демо-данные реальными.
2. Подтвердить рабочий URL квиза (пока веду на `my.makeunion.me/simulators/ozon-quiz`).
3. Логотип/аватар в фирменном виде (сейчас взял с сайта эталона) — если есть отдельные файлы.
4. Домен и аккаунт для публикации (деплой сделаю по вашей отмашке).

**Риски:** без доступа к реальной базе каталог остаётся на демо-данных; финальный визуал ещё дорабатываю под эталон.

**Следующий шаг:** доведу визуал и, по готовности доступов, подключу реальные данные и задеплою, чтобы дать рабочую ссылку.
